### Fixed

- **Reverse search worker idle / no logs:** `POST /reverse-search` returned **cache_hit** for rows whose **`web_results` was `[]`**, so **no job was enqueued** and the worker only logged startup. Cached empty results are no longer treated as hits (same provider + prompt_version); **`reverse_search_job_claimed`** / **`reverse_search_job_completed`** appear once jobs flow. API logs **`reverse_search_submit_external`**; worker logs **`database_hint`** on start, **`reverse_search_worker_waiting_for_jobs`** after ~10s idle, and **`reverse_search_job_claimed`** per job.
- **Composite reverse search (Apify Pinterest):** default actor `epctex~pinterest-search-scraper` was invalid store-side and inputs used `searchQueries` / `resultsLimit`, which Pinterest search actors do not accept — runs returned empty datasets. Defaults now use **`scraperforge~pinterest-search-scraper`**-shaped input (`query`, `filter`, `limit`), up to four queries in parallel, deduped rows, **`node_id` → pin URL** when `link` is blank, and **`reaction_counts`** engagement. Parallel runs use **`return_exceptions=True`** so one failing sub-query no longer wipes the entire Pinterest chunk. Instagram composite calls pass **`resultsType`: `posts`**.
- **Fit Score worker:** result-insert no longer crashes on the FK column. `mirror-fit-score-worker` was passing `asyncpg.Record["product_id"]` (a `uuid.UUID` object) straight into the `sb.table("fit_score_results").insert(...)` payload, which `json.dumps` rejected — every job ended up `status='failed'`, `error_code='INTERNAL'`. Now coerced to `str()` to match the `id` / `user_id` discipline already used in the same function and in `avatar_worker`. ([`apps/backend/src/mirror/workers/fit_score_worker.py`](apps/backend/src/mirror/workers/fit_score_worker.py))
- **Worker logging:** `structlog` JSON renderer now includes `format_exc_info`, so `log.exception(...)` lines (the catch-all in every worker loop) actually render the traceback instead of just `"exc_info": true`. ([`apps/backend/src/mirror/core/logging_config.py`](apps/backend/src/mirror/core/logging_config.py))
- **Vercel web:** [`apps/web/vercel.json`](apps/web/vercel.json) pins monorepo **install** / **build** commands; [docs/06 §4](docs/06_Deployment_Operations.md) corrected **Output Directory** guidance (leave empty — never `public`) to avoid _No Output Directory named ‘public’_ deployment failures.
- Extension **Feed** tab: redesigned the side-panel feed to match the `phia_mobile` editorial system more closely, with warmer paper/card styling, serif captions, cleaner post metadata, and a new inline **share** action in the card footer.
- Extension try-on: PDP garment bucket (**top** vs **bottom**) is inferred from product title, URL path, and optional JSON-LD `category`, so shorts/pants/skirts default to the **bottom** slot instead of always starting as **top**.
- Extension side panel **Feed** tab: shows **only the signed-in user’s** recent approved posts (limit 8), **author** line uses profile/email (not hardcoded “Community”), **reaction/comment** counts on each card, **fresh `tryon-results` signed URLs** on each load so images work after closing and reopening the extension, and **“Open web to see more”** when `VITE_MIRROR_WEB_URL` is set.
- Web `/feed`: post images load via **fresh signed URLs** from `tryon_results.storage_path` using a **server-only** Supabase service client (fixes **expired** `posts.image_url` and **tryon-results** storage RLS for other users’ posts). Realtime inserts refresh via **POST `/api/feed/signed-urls`**.
- Backend: `Settings.api_port` now prefers **`PORT` over `API_PORT`** when both are set, so Railway deploys are not broken by a stray `API_PORT=8000` variable (fixes repeated `/health` “service unavailable” during deploy).
